Digital IOs NodeEEBench

Use cases

  1. An ADC, sensor is connected via SPI or I2C
  2. A DAC, actor is connected via SPI or I2C
  3. SPI, I2C, serial transmit and receive
  4. Configuring pins as inputs, outpus, bidirectional
  5. Comparision and pass/fail information
  6. An 8-bit R2R DAC is connected to 8 outputs
  7. A serial DAC with one serial data line and 3 clock signals is connected
  8. A pipeline ADC gives a n-Bit serial input data (gray code) signal and needs 3/4 output clock signals
  9. A SAR ADC generates one compare input and sets a serial/parallel DAC accordingly
  10. The AWG output sends binary values to digital outputs using a lookup table
  11. A serial coded signal is received, passed through a lookup table, a signal processing unit and displayed with the oscilloscope or fed to the AWG

Implementation Considerations

  1. Digital data acquisition should replace analog data at acquisition with a certain block size
  2. Digital data can be displayed as single bit trace, hex numbers or analog values (coding: binary, gray code, signed)
  3. It is difficult to configure individual pins with the Arduino.
    There should be dedicated configurations for input, output, serial, parallel pins.
  4. Arduino digital IOs:
    A4 serial output, A5 serial input, D0..D7 outputs, inputs, half/half
    I2C: SCL (D12), SDA(D11)
    SPI: MOSI (D8), MISO(D10), CLK (D9), CS (A6)
  5. Example implementation could be
    (1) R2R,
    (2) serial DAC,
    (3) SAR (cmp signal serial input), SAR algorithmic calculation with serial DAC (data signal serial output + clk and cntrl pins)
    Low pass
